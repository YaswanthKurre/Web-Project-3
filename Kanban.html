Kanban and DevOps Methodology

Kickoff and Idea Gathering

  • Start by understanding what the 15-puzzle game should look like.
  • Create a to-do list of tasks on Excel or a digital tool.

Breaking Down Tasks

  • Break the game creation into smaller jobs you can handle easily.
  • Prioritize tasks based on what's most important for the game to start with.

Setting Up a Project Board

  • Make a simple board with columns like "To-Do," "Doing," and "Done".
  • Use Excel or a digital tool to track your progress.

Coding Parts of the Game

  • Begin writing code for different parts of the game separately.
  • Work on coding the game rules, the visual part, and how the puzzle gets created.

Putting Code Together

  • Use a step-by-step approach to combine your code snippets into one game.
  • Test each part as you go to catch any issues early on.

Checking Each Other's Work

  • Share your code with teammates for a quick review.
  • Give and receive feedback to make your code better.

Testing the Game

  • Combine all the code snippets to make a simple version of the game.
  • Test it to make sure everything works together.

Getting Feedback

  • Show your game to friends for their thoughts.
  • Use their feedback to make improvements.

Making It Better

  • Keep improving your game based on the feedback.
  • Add extra features or make changes to make it more fun.
  • Get your game ready for the final presentation.
